FSM 转换规则 (来自 D-01, D-02):
REASON --[有 tool_calls]--> ACT
REASON --[无 tool_calls，有 content]--> FINISH (D-01)
REASON --[LLM 错误]--> ERROR
ACT --[工具执行成功]--> OBSERVE
ACT --[工具执行错误]--> ERROR
OBSERVE --[步数 < 最大]--> REASON
OBSERVE --[步数 >= 最大]--> FINISH (通过 BudgetGuard "final")
OBSERVE --[不可达]--> FINISH
ANY --[未处理异常]--> ERROR
ERROR: Phase 1 的终止状态 (RESEARCH.md A1)

守卫集成:
- REASON 前: BudgetGuard.check() + MessageValidator.validate()
- ACT 前: LoopDetector.check()
- OBSERVE 中: BudgetGuard.check_unreachable()

Phase 1 工具处理: 目前尚无工具 (Phase 2 才构建)。当 LLM 返回 tool_calls 时，FSM 以合成 tool_result 转换 ACT->OBSERVE: "[SYSTEM] No tools are available in Phase 1. Please provide your answer directly." 这样可以在没有实际工具执行的情况下演练完整循环。

优雅关闭模式:
1. 运行 FSM 到完成 (FINISH 或 ERROR)
2. 通过 bus.shutdown() 向所有订阅者队列发送 None 哨兵
3. 使用 asyncio.wait_for(timeout=5.0) 等待消费者任务排空
4. 调用 logger.stop() 执行 fsync + 关闭
