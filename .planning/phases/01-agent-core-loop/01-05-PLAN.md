FSM Transition Rules (from D-01, D-02):
REASON --[has tool_calls]--> ACT
REASON --[no tool_calls, has content]--> FINISH (D-01)
REASON --[LLM error]--> ERROR
ACT --[tool executed OK]--> OBSERVE
ACT --[tool execution error]--> ERROR
OBSERVE --[step < max]--> REASON
OBSERVE --[step >= max]--> FINISH (via BudgetGuard "final")
OBSERVE --[unreachable]--> FINISH
ANY --[unhandled exception]--> ERROR
ERROR: terminal for Phase 1 (RESEARCH.md A1)

Guard integration:
- Pre-REASON: BudgetGuard.check() + MessageValidator.validate()
- Pre-ACT: LoopDetector.check()
- In OBSERVE: BudgetGuard.check_unreachable()

Phase 1 Tool Handling: No tools exist yet (built in Phase 2). When LLM returns tool_calls, FSM transitions ACT->OBSERVE with synthetic tool_result: "[SYSTEM] No tools are available in Phase 1. Please provide your answer directly." This exercises the full cycle without real tool execution.

Graceful Shutdown Pattern:
1. Run FSM to completion (FINISH or ERROR)
2. Send None sentinel to all subscriber queues via bus.shutdown()
3. Await consumer tasks drain with asyncio.wait_for(timeout=5.0)
4. Call logger.stop() for fsync + close
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI input -> Agent | User prompt enters via argparse. Untrusted input crosses into LLM context. |
| LLM -> FSM | LLM response crosses into state machine. Validated by MessageValidator before next call. |
| FSM -> Consumers | Events published to EventBus for display and logging. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | main.py | mitigate | API key never printed; verbose mode does NOT log api_key; SecretStr prevents accidental exposure |
| T-05-02 | Denial of Service | ReActFSM | mitigate | BudgetGuard enforces max_steps; LoopDetector blocks infinite loops; shutdown timeout 5s |
| T-05-03 | Spoofing | ReActFSM._handle_act | mitigate | Tool results wrapped in tool-role messages with tool_call_id; never merged into user messages |
| T-05-04 | Elevation of Privilege | ReActFSM._handle_act | mitigate | LLM tool_calls.function.arguments parsed ONLY via json.loads(); NEVER eval()/exec(); Phase 1 no-ops all tools |
| T-05-05 | Repudiation | JSONLLogger | accept | JSONL log provides audit trail; no cryptographic signing in Phase 1 |
</threat_model>
