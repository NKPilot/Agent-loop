<interfaces>
BudgetGuard (来自 RESEARCH.md 模式 5 + D-06, D-07, D-09):
- 构造函数: max_steps: int = 15, warn_pct: float = 0.80
- 方法: check(step_count: int, messages: list[dict]) -> tuple[bool, list[dict], str | None]
- 返回: (should_continue, modified_messages, action)，其中 action 值为 None/"warn"/"final"
- 在警告阈值 (80%): 注入包含百分比和剩余步数的系统消息
- 在耗尽时: 注入最终摘要提示，返回 action="final"
- D-08 不可达检测: 系统检测到连续失败 >= 3 -> 发出不可达信号

LoopDetector (来自 RESEARCH.md 模式 4 + CORE-06):
- 构造函数: window_size: int = 20, warn_threshold: int = 3, block_threshold: int = 5
- 静态方法: _signature(tool_name: str, arguments: dict) -> str 使用 hashlib.sha256
- 方法: check(tool_name: str, arguments: dict) -> tuple[bool, str] 返回 (should_proceed, action)
- action 取值: "allow", "warn", "block" (连续 5 次时), "force_exit" (封锁后模式仍持续)

MessageValidator (来自 RESEARCH.md + CORE-05):
- 严格验证，按 RESEARCH 开放问题 #2 的建议
- 检查: 每个带 tool_calls 的 assistant 消息都有对应的 tool-role 消息来匹配每个 tool_call_id
- 违规时: 抛出 ValidationError，并指明哪个 tool_call_id 是孤立的

AgentConfig (Claude 自行裁定 + RESEARCH 开放问题 #1):
- 主要来源: 环境变量 OPENAI_API_KEY, OPENAI_BASE_URL (默认 https://api.openai.com/v1), OPENAI_MODEL (默认 gpt-4o)
- 次要来源: CLI 标志覆盖环境变量
- Pydantic 模型，使用 SecretStr 存储 api_key

## 信任边界

| 边界 | 描述 |
|------|------|
| 环境 -> Config | 不受信任的环境变量值成为 AgentConfig 字段。pydantic 验证约束类型。 |
| LLM 输出 -> MessageValidator | 来自 LLM 的消息 (tool_calls 结构) 在下一次 API 调用前验证。 |

## STRIDE 威胁登记表

| 威胁 ID | 类别 | 组件 | 处置方式 | 缓解计划 |
|---------|------|------|---------|---------|
| T-02-01 | 信息泄露 | AgentConfig.api_key | 缓解 | pydantic 模型中使用 SecretStr；自定义 __repr__ 隐藏值；永不记录或序列化 |
| T-02-02 | 拒绝服务 | BudgetGuard | 缓解 | step_count >= max_steps 强制 "final" action，无论 LLM 行为如何 |
| T-02-03 | 欺骗 | MessageValidator | 缓解 | 严格的孤立检测防止在没有对应 tool_call 的情况下注入 tool_result |
| T-02-04 | 权限提升 | LoopDetector._signature | 接受 | sha256 仅用于确定性比较，非安全用途；来自 LLM 工具调用的 args 受信任 |
</threat_model>
