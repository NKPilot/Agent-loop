From RESEARCH.md Architecture Patterns:

Pattern 2 (EventBus): asyncio.Queue-based pub/sub. Each subscriber gets its own bounded queue (maxsize=256). Publish fans out to all subscriber queues. Consumers are long-running async tasks that `await queue.get()` in a loop. Shutdown via None sentinel.

Pattern 3 (Layered Events): Two-level hierarchy. Top-level lifecycle (step_start, step_end, session_end) wrap inner streaming events (llm_token, tool_call_start, tool_call_args, tool_call_done, tool_result).

Pattern 3 event types (detailed):
- StepStart: { session_id, step_num, timestamp }
- StepEnd: { session_id, step_num, timestamp, state_transition, token_usage }
- SessionEnd: { session_id, timestamp, final_state, total_steps, exit_reason }
- LLMToken: { session_id, step_num, content_delta }
- LLMContentDone: { session_id, step_num, full_content }
- ToolCallStart: { session_id, step_num, tool_name, tool_call_id }
- ToolCallArgs: { session_id, step_num, tool_name, args_delta }
- ToolCallDone: { session_id, step_num, tool_name, tool_call_id, full_args }
- ToolResult: { session_id, step_num, tool_name, tool_call_id, result, is_error, duration_ms }
- BudgetWarning: { session_id, step_num, used_pct, max_steps }
- BudgetExhausted: { session_id, step_num }
- LoopDetected: { session_id, step_num, tool_name, consecutive_count }
- Error: { session_id, step_num, error_type, message, traceback }

- D-03: Dual-granularity events (step-level + token-level)

  <name>Task 3: Implement EventBus with asyncio.Queue pub/sub per D-05</name>
  <files>src/loopai/events/bus.py, tests/test_event_bus.py</files>
  <read_first>src/loopai/events/schemas.py (for Event union type and model_dump_json usage)</read_first>
  <action>
    Implement src/loopai/events/bus.py with class EventBus:

    CONSTRUCTOR:
    - _subscribers: dict[str, list[asyncio.Queue]] — maps topic (event_type string or "*") to subscriber queues
    - _history: list[dict] — all published events for replay
    - _lock: asyncio.Lock — guards _subscribers during subscribe/unsubscribe

    PUBLISH (event_type: str, event_data: dict) -> None:
    - Append event_data to _history
    - For each queue in _subscribers.get(event_type, []) + _subscribers.get("*", []): await queue.put(event_data)
    - If a queue is full (QueueFull exception from put_nowait), log a warning via warnings.warn and drop the event for that slow consumer -- do NOT block the publisher
    - Validate event_data is JSON-serializable by calling json.dumps(event_data) before publishing (raises TypeError if non-serializable)
    - Each event_data must contain an "event_type" key matching the publish topic

    SUBSCRIBE (topic: str) -> asyncio.Queue:
    - Create bounded asyncio.Queue(maxsize=256)
    - Under lock, append to _subscribers[topic]
    - Return the queue

    UNSUBSCRIBE (topic: str, queue: asyncio.Queue) -> None:
    - Under lock, remove queue from _subscribers[topic]
    - Clean up empty topic lists

    REPLAY (topic: str | None = None) -> list[dict]:
    - If topic is None, return all _history
    - If topic is "*" wildcard, return all _history
    - Otherwise return [e for e in _history if e["event_type"] == topic]

    SHUTDOWN:
    - publish sentinel: put None into every subscriber queue
    - Consumers detect None and exit their loop

    Then create tests/test_event_bus.py with these test cases:
    - test_publish_subscribe_single_topic: subscribe to "llm_token", publish, verify queue.get() returns the event
    - test_fan_out_multiple_subscribers: subscribe two queues to same topic, publish once, verify both queues receive the event
    - test_wildcard_subscription: subscribe with "*", publish to "step_start", verify wildcard subscriber receives it AND topic subscriber does not get it (no topic subscriber)
    - test_replay_history: publish 3 events, verify replay() returns all 3 in order
    - test_replay_by_topic: publish events of 2 types, verify replay("llm_token") returns only llm_token events
    - test_unsubscribe: subscribe, unsubscribe, publish, verify queue.get() would block (use asyncio.wait_for with 0.1s timeout)
    - test_shutdown_sentinel: subscribe, shutdown (send None), verify queue.get() returns None
    - test_bounded_queue_backpressure: subscribe, set queue maxsize=2, publish 3 events without consumer draining, verify no deadlock (third publish drops with warning, not blocks)
    - test_invalid_event_data: publish non-serializable data (e.g., a set), verify TypeError is raised before queue.put

    Run `python -m pytest tests/test_event_bus.py -v` -- all 9 tests must pass.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Event publisher -> EventBus | Untrusted event data crosses into the bus at publish(). Validated for JSON-serializability. |
| EventBus -> Consumer queue | Events delivered to subscriber queues. Consumers must handle potentially slow drains. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | pyproject.toml, config | mitigate | API key read from env var only (OPENAI_API_KEY); never hardcoded in any file; .env and .env.* in .gitignore |
| T-01-02 | Denial of Service | EventBus.publish() | mitigate | Bounded queues (maxsize=256); QueueFull triggers warning + drop, not deadlock; prevents unbounded memory growth |
| T-01-03 | Tampering | EventBus.publish() | mitigate | Validate event_data is JSON-serializable at publish time via json.dumps() call; reject non-serializable data with TypeError |
| T-01-04 | Information Disclosure | Event schemas | accept | Event models may contain tool output data; JSONL logger will handle permissions (0o600) in Plan 04; no PII expected at this layer |
</threat_model>
