来自 RESEARCH.md 架构模式:

模式 2 (EventBus): 基于 asyncio.Queue 的发布/订阅。每个订阅者拥有独立的有界队列 (maxsize=256)。发布操作将事件扇出到所有订阅者队列。消费者是长时间运行的异步任务，在循环中 `await queue.get()`。通过 None 哨兵值实现关闭。

模式 3 (分层事件): 两级层次结构。顶层生命周期事件 (step_start, step_end, session_end) 包裹内部流式事件 (llm_token, tool_call_start, tool_call_args, tool_call_done, tool_result)。

模式 3 事件类型 (详细):
- StepStart: { session_id, step_num, timestamp }
- StepEnd: { session_id, step_num, timestamp, state_transition, token_usage }
- SessionEnd: { session_id, timestamp, final_state, total_steps, exit_reason }
- LLMToken: { session_id, step_num, content_delta }
- LLMContentDone: { session_id, step_num, full_content }
- ToolCallStart: { session_id, step_num, tool_name, tool_call_id }
- ToolCallArgs: { session_id, step_num, tool_name, args_delta }
- ToolCallDone: { session_id, step_num, tool_name, tool_call_id, full_args }
- ToolResult: { session_id, step_num, tool_name, tool_call_id, result, is_error, duration_ms }
- BudgetWarning: { session_id, step_num, used_pct, max_steps }
- BudgetExhausted: { session_id, step_num }
- LoopDetected: { session_id, step_num, tool_name, consecutive_count }
- Error: { session_id, step_num, error_type, message, traceback }

  <action>
    实现 src/loopai/events/bus.py，包含 EventBus 类:

    构造函数:
    - _subscribers: dict[str, list[asyncio.Queue]] -- 将主题 (event_type 字符串或 "*") 映射到订阅者队列列表
    - _history: list[dict] -- 所有已发布的事件，用于回放
    - _lock: asyncio.Lock -- 在 subscribe/unsubscribe 期间守卫 _subscribers

    PUBLISH (event_type: str, event_data: dict) -> None:
    - 将 event_data 追加到 _history
    - 对于 _subscribers.get(event_type, []) + _subscribers.get("*", []) 中的每个队列: await queue.put(event_data)
    - 如果队列已满 (put_nowait 引发 QueueFull 异常)，通过 warnings.warn 记录警告并为该慢速消费者丢弃事件 -- 不要阻塞发布者
    - 发布前通过调用 json.dumps(event_data) 验证 event_data 是 JSON 可序列化的 (不可序列化则引发 TypeError)
    - 每个 event_data 必须包含与发布主题匹配的 "event_type" 键

    SUBSCRIBE (topic: str) -> asyncio.Queue:
    - 创建有界 asyncio.Queue(maxsize=256)
    - 在锁内追加到 _subscribers[topic]
    - 返回该队列

    UNSUBSCRIBE (topic: str, queue: asyncio.Queue) -> None:
    - 在锁内从 _subscribers[topic] 移除 queue
    - 清理空的主题列表

    REPLAY (topic: str | None = None) -> list[dict]:
    - 如果 topic 为 None，返回所有 _history
    - 如果 topic 为 "*" 通配符，返回所有 _history
    - 否则返回 [e for e in _history if e["event_type"] == topic]

    SHUTDOWN:
    - 发布哨兵: 将 None 放入每个订阅者队列
    - 消费者检测到 None 后退出其循环

    然后创建 tests/test_event_bus.py，包含以下测试用例:
    - test_publish_subscribe_single_topic: 订阅 "llm_token"，发布，验证 queue.get() 返回该事件
    - test_fan_out_multiple_subscribers: 订阅两个队列到同一主题，发布一次，验证两个队列都收到事件
    - test_wildcard_subscription: 用 "*" 订阅，向 "step_start" 发布，验证通配符订阅者接收到事件且主题订阅者不会收到 (没有主题订阅者)
    - test_replay_history: 发布 3 个事件，验证 replay() 按顺序返回全部 3 个
    - test_replay_by_topic: 发布 2 种类型的事件，验证 replay("llm_token") 仅返回 llm_token 事件
    - test_unsubscribe: 订阅、取消订阅、发布，验证 queue.get() 会阻塞 (使用 asyncio.wait_for 设置 0.1s 超时)
    - test_shutdown_sentinel: 订阅、shutdown (发送 None)，验证 queue.get() 返回 None
    - test_bounded_queue_backpressure: 订阅，设置 queue maxsize=2，不消费队列的情况下发布 3 个事件，验证不会死锁 (第三个发布被丢弃并发出警告，而非阻塞)
    - test_invalid_event_data: 发布非可序列化数据 (如 set)，验证在 queue.put 之前引发 TypeError
