| Boundary | Description |
|----------|-------------|
| LLMClient -> OpenAI API | API key crosses network boundary via HTTPS. |
| OpenAI API response -> LLMClient | Untrusted response data crosses into agent. Validated by EventBus at publish time. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | LLMClient | mitigate | api_key extracted via SecretStr.get_secret_value() only at call site; not stored in instance attributes; not included in any published events |
| T-03-02 | Spoofing | LLMClient.complete() | accept | tool_calls in LLM response trusted as-is; Phase 2 adds tool parameter validation |
| T-03-03 | Denial of Service | LLMClient.complete() | mitigate | Streaming timeout via httpx SDK defaults + BudgetGuard step limits |
| T-03-04 | Elevation of Privilege | LLMClient | mitigate | LLM output NEVER eval()'d or exec()'d; arguments parsed via json.loads() only |
</threat_model>
