## 信任边界

| 边界 | 描述 |
|------|------|
| LLMClient -> OpenAI API | API key 通过 HTTPS 跨越网络边界。 |
| OpenAI API 响应 -> LLMClient | 不受信任的响应数据进入 agent。在发布时由 EventBus 验证。 |

## STRIDE 威胁登记表

| 威胁 ID | 类别 | 组件 | 处置方式 | 缓解计划 |
|---------|------|------|---------|---------|
| T-03-01 | 信息泄露 | LLMClient | 缓解 | api_key 仅在调用点通过 SecretStr.get_secret_value() 提取；不存储在实例属性中；不包含在任何发布的事件中 |
| T-03-02 | 欺骗 | LLMClient.complete() | 接受 | LLM 响应中的 tool_calls 按原样信任；Phase 2 添加工具参数验证 |
| T-03-03 | 拒绝服务 | LLMClient.complete() | 缓解 | 通过 httpx SDK 默认值流式超时 + BudgetGuard 步数限制 |
| T-03-04 | 权限提升 | LLMClient | 缓解 | LLM 输出永不 eval() 或 exec()；参数仅通过 json.loads() 解析 |
