## Color

All color values reference shadcn/ui v4 CSS variables (HSL). The executor will use Tailwind v4 utilities that resolve to these variables.

| Role | CSS Variable | Approximate Hex (Light) | Usage |
|------|-------------|------------------------|-------|
| Dominant (60%) | `hsl(var(--background))` | `#ffffff` | Page background, panel backgrounds |
| Secondary (30%) | `hsl(var(--card))` / `hsl(var(--muted))` | `#ffffff` / `#f4f4f5` | Cards, sidebar, navigation, muted sections |
| Accent (10%) | `hsl(var(--primary))` | `#18181b` | Primary text, active indicators, focus rings, CTA buttons |
| Destructive | `hsl(var(--destructive))` | `#ef4444` | Delete session button, error badges, rejection actions |
| Success | Custom token: `#16a34a` | `#16a34a` | Completed step badge, successful tool result indicator |
| Warning | Custom token: `#d97706` | `#d97706` | Budget warning badge, loop detected indicator |
| Info | `hsl(var(--accent))` | `#f4f4f5` | Inactive/secondary states, hover backgrounds |

**Accent reserved for:**
- Primary CTA button fills (e.g., "Start Agent")
- Active session indicator dot (green variant)
- Selected sidebar item background
- Focus ring outlines (`ring-primary`)
- Step numbers on active steps
- Link text and hover states on interactive elements

**Accent NOT used for:**
- General text (use foreground)
- Card borders (use border)
- Background fills (use background/card/muted)
- Badge fills (use muted or semantic tokens)

**Semantic badge color mapping:**

| Event Type / State | Badge Variant | Example |
|-------------------|---------------|---------|
| Running / In Progress | `default` (primary bg) | "Running" on active session |
| Completed / Success | `success` (green) | "Done" on completed tool call |
| Error / Failed | `destructive` | "Error" on failed tool call |
| Warning / Budget | `warning` (amber) | "80% Budget" warning |
| Paused / Waiting | `secondary` (muted) | "Awaiting Confirmation" |

Source: shadcn/ui default theme + two custom semantic tokens (success green, warning amber) defined for event state clarity. The success and warning tokens are Claude's Discretion additions necessary for event type differentiation.

## Layout Contract

### Three-Panel Structure (Decision D-01)

```
+------------------+------------------------+------------------+
| Session List     | Agent Timeline        | Tool Detail      |
| (Left, 260px)    | (Center, flex-1)      | (Right, 360px)   |
|                  |                        |                  |
| [Search sessions]| Step 1: REASON        | Tool: disk_usage |
|                  |   "Let me analyze..." | Status: done     |
| Session #1       |                        | Duration: 1.2s   |
| Session #2  (*)  | Step 2: ACT           |                  |
| Session #3       |   disk_usage()        | Args: {...}      |
| Session #4       |   Status: running     | Result: {...}    |
|                  |                        |                  |
| Session #5       | Step 3: OBSERVE       | Token: 450       |
|                  |   "Result shows..."   | Cost: $0.003     |
+------------------+------------------------+------------------+
```

**Panel constraints:**
- Left panel: fixed 260px width, scrollable session list
- Center panel: flex-1 (fills remaining space), scrollable timeline with auto-scroll-to-bottom
- Right panel: fixed 360px width, scrollable detail view
- Minimum viewport width: 1024px (below this, panels collapse/stack)
- Panel separators: 1px `border-border` vertical divider

### Responsive Breakpoints

| Breakpoint | Behavior |
|-----------|----------|
| >= 1280px | Full three-panel layout |
| 1024px - 1279px | Right panel collapses to bottom or becomes a drawer |
| < 1024px | Single-column stack: sessions -> timeline -> detail |

### Session List (Left Panel)

- Search/filter input at top
- Session list items show: session ID (truncated), step count, status badge, timestamp
- Active session highlighted with accent left-border (2px)
- Click to switch active session
- Each item has a context menu (via DropdownMenu): Delete, Export JSONL
- Empty state shows when no sessions exist

### Agent Timeline (Center Panel)

- Chronological list of steps (StepStart ... StepEnd pairs)
- Each step shows:
  - Step number badge (circular, 24px)
  - Step type: REASON (blue tint), ACT (amber tint), OBSERVE (green tint)
  - Streaming text content (from LLMToken events, rendered incrementally)
  - Tool call mini-cards inline (name + status badge)
  - Guard events inline (budget warning, loop detected) as Alert variants
- Auto-scroll to bottom on new events (with "scroll to bottom" button if user scrolled up)
- Step states: `pending` (dimmed), `active` (pulsing dot), `completed`, `error` (red border)

### Tool Detail (Right Panel)

- Shows when a ToolCallStart event is received for the active session
- Content sections:
  - Header: tool name + status badge + duration
  - Arguments: JSON display (syntax-highlighted, monospace)
  - Result: text content (or truncated with overflow file reference)
  - Metadata: tool_call_id, token consumption, cost estimate
- Transitions: slides content when a new tool call starts
- Empty state: "Select a tool call to view details"

Source: Decision D-01 from CONTEXT.md, component mapping is Claude's Discretion.

## Interaction Contract

### SSE Connection (Decisions D-03, D-04)

| State | Visual |
|-------|--------|
| Connected | Green dot + "Live" badge in header |
| Connecting | Amber dot + "Connecting..." badge |
| Disconnected (retrying) | Red dot + "Reconnecting (attempt N)..." badge + Alert banner |
| Disconnected (failed) | Red dot + "Connection Failed" badge + Alert banner + "Retry Now" button |

- Connection indicator lives in the top header bar
- Reconnection countdown shows in the Alert banner body
- Max reconnection attempts: 10 (after which user must manually retry)

### Streaming Content Rendering

- LLMToken deltas append to the current reasoning block in real-time
- ToolCallArgs deltas append to a "building arguments..." display
- Cursor blinks at end of streaming content
- Completed content transitions from italic to normal weight
- Content block transitions use CSS `transition: opacity 150ms ease`

### Confirmation Dialog (Decision D-06)

| Element | Value |
|---------|-------|
| Dialog title | "Dangerous Command Detected" |
| Icon | `AlertTriangle` (lucide-react) in destructive color |
| Body | Shows tool name, full command text (monospace, scrollable), reason for flagging |
| Approve button | "Approve" -- solid destructive variant |
| Reject button | "Reject" -- outline variant |
| Auto-timeout | None -- user must explicitly choose |
| After action | Dialog closes, timeline shows the confirmation response event |

### Session Selection

- Click session in left panel -> load its timeline in center
- Only one session active at a time
- Switching from a live session shows a "Viewing history" banner (live session continues in background)
- Live session shows real-time updates; historical sessions show static snapshots

### Keyboard Navigation

| Key | Action |
|-----|--------|
| `j` / `ArrowDown` | Move to next session in list |
| `k` / `ArrowUp` | Move to previous session in list |
| `Enter` | Open selected session |
| `Escape` | Close confirmation dialog |
| `t` | Switch to Timeline tab |
| `u` | Switch to Token Usage tab |
| `r` | Switch to Raw Events tab |

Source: Claude's Discretion (D-05 lists full feature delivery but leaves interaction specifics open).

## Token / Cost Display Contract

### Token Usage Summary Card

Displayed in the right panel or as an overlay. Shows:

| Metric | Format | Source Event |
|--------|--------|-------------|
| Prompt tokens | `1,234` (comma-separated) | StepEnd.token_usage.prompt_tokens |
| Completion tokens | `567` | StepEnd.token_usage.completion_tokens |
| Total tokens (session) | `12,345` | Cumulative from all StepEnd events |
| Estimated cost | `$0.042` | Calculated: prompt * $0.003/1K + completion * $0.015/1K (default rates based on gpt-4o-mini; configurable) |
| Context window usage | `45%` with progress bar | TokenWarning.used_pct or cumulative tracking |

### Token Warning Display

When TokenWarning event received:
- Amber badge in timeline: "Token 82%"
- Progress bar turns amber at 75%, red at 90%
- "Context Compacting..." message appears when ContextCompacted event fires

### Cost Tracking Chart

recharts AreaChart showing cumulative cost ($) over session steps (X-axis: step number, Y-axis: cumulative cost in USD).

## State Management Architecture

| Concern | Tool | What It Manages |
|---------|------|----------------|
| Server state | @tanstack/react-query 5 | Session list (GET /api/sessions), session detail/events (GET /api/sessions/{id}), SSE stream as query invalidation source |
| Client state | Zustand 5 | Active session ID, selected tool call ID, panel collapse state, theme preference (future), filter settings |
| Real-time updates | SSE + `queryClient.setQueryData()` | Optimistic updates to timeline and tool call data as events stream in |

### SSE Event -> React Query Flow

```
EventBus (Python) --> SSE Bridge --> EventSource (frontend)
  --> onEvent(event):
    switch event.event_type:
      case "step_start": append step to query cache
      case "llm_token": append delta to current reasoning block
      case "tool_call_start": create tool card placeholder
      case "tool_result": update tool card with result + duration
      case "step_end": finalize step with token_usage
      case "session_end": mark session complete, invalidate session list
      case "error": show error in timeline + alert banner
      case "confirmation_required": open Dialog, pause timeline
```

Source: Decision D-02 (tech stack) from CONTEXT.md. Event-to-UI mapping is Claude's Discretion based on the existing 20-event schema from `src/loopai/events/schemas.py`.

## API Contract (Frontend View)

| Endpoint | Method | Returns | Used By |
|----------|--------|---------|---------|
| `/api/sessions` | GET | `{ sessions: [{ id, created_at, step_count, status, exit_reason }] }` | Session list (left panel) |
| `/api/sessions/{id}` | GET | `{ session: { id, steps, events, token_usage, cost } }` | Session detail / historical view |
| `/api/sessions/{id}/stream` | GET (SSE) | `text/event-stream` of Event objects | Real-time timeline (center panel) |
| `/api/sessions/start` | POST | `{ session_id }` | "Start Agent" button |
| `/api/sessions/{id}/confirm/{confirmation_id}` | POST | `{ approved: bool }` | Confirmation dialog response |
| `/api/sessions/{id}` | DELETE | `{ deleted: true }` | Delete session |
| `/api/sessions/{id}/export` | GET | `application/jsonl` download | Export session JSONL |

Source: Derived from OBS-01 through OBS-05 requirements and existing EventBus architecture.
