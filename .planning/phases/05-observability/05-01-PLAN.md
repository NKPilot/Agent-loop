class EventBus:
    async def publish(event_type: str, event_data: dict) -> None
    async def subscribe(topic: str) -> asyncio.Queue[dict | None]

    1. 创建 `src/loopai/api/sse_bridge.py`：实现 `async def event_stream(session_id: str, bus: EventBus) -> AsyncIterable[ServerSentEvent]`：
       - 使用 `queue = await bus.subscribe("*")` 订阅所有事件
       - REPLAY 阶段：遍历 `bus.replay()` 返回的历史事件，过滤 `event["session_id"] == session_id`，为每个事件 yield `ServerSentEvent(data=event, event=event["event_type"], id=str(len(event)), retry=3000)`。按 RESEARCH.md 建议限制 replay 只返回最近 500 个事件避免内存压力（截取 `bus.replay()[-500:]`）
       - STREAM 阶段：`while True` 循环从 queue 获取事件，如果事件为 None（shutdown sentinel）则 break。过滤 `event.get("session_id") == session_id`，yield `ServerSentEvent(data=event, event=event["event_type"], id=str(seq_counter), retry=3000)`。使用递增的 seq_counter（与 replay 的 len 分开，从 replay 后的 count 开始）
       - 使用 `try/finally` 确保 disconnect 时调用 `await bus.unsubscribe("*", queue)`
       - IMPORTANT：不要在循环内做任何 I/O 操作——只做过滤和 yield（Pitfall 1 避免背压）
       - 事件 data 是 dict —— FastAPI ServerSentEvent 会自动 JSON 序列化（通过 `json.dumps` 在内部）

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→SSE endpoint | 不受信任的 HTTP 请求到达 `/api/sessions/{id}/stream` |
| EventBus→SSE bridge | EventBus 内部数据流向外部 HTTP 客户端 |
| session_id path param | 用户提供的路径参数，用于过滤事件和构造日志路径 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | sse_bridge.py:event_stream() | mitigate | `bus.replay()` 返回所有事件——SSE bridge 必须在 replay 阶段按 `session_id` 过滤，防止跨会话数据泄露。已在 `<action>` 中指定过滤逻辑。 |
| T-05-02 | Information Disclosure | sse_bridge.py:event_stream() | mitigate | SSE stream 阶段也需按 `session_id` 过滤。`event.get("session_id")` 检查防止将其他 session 的事件泄露给当前客户端。 |
| T-05-03 | Denial of Service | app.py:create_app() | accept | 无身份验证——v1 本地单用户工具，localhost-only 部署。仅允许 localhost origins 的 CORS 策略作为基本边界控制。 |
| T-05-04 | Elevation of Privilege | routes/stream.py | accept | session_id 路径参数无授权检查——任何知道 session_id 的客户端都可以监听。v1 本地工具范围内可接受。 |
</threat_model>
