  <action>
    1. 在项目根目录创建 `frontend/` 目录。进入目录运行 `pnpm create vite . --template react-ts` 创建 Vite 8 + React 19 + TypeScript 项目。
    2. 安装依赖（D-02 锁定的库）：`pnpm add @tanstack/react-query@5 zustand recharts lucide-react`
    3. 配置 Tailwind 4：Tailwind 4 使用 CSS-first 配置，无需 tailwind.config.js。确认 `@tailwindcss/vite` 插件已在 package.json 中（Vite 模板自带）。
    4. 在 `frontend/src/styles/index.css` 顶部写入：`@import "tailwindcss";`。在 CSS 中定义自定义语义颜色令牌（per UI-SPEC Color section）：
       ```css
       @theme {
         --color-success: #16a34a;
         --color-warning: #d97706;
       }
       ```
       字体栈使用 Inter（shadcn default）。
    5. 初始化 shadcn/ui CLI v4：运行 `npx shadcn@latest init` 并传递以下参数以非交互模式运行：
       - 如果 CLI 支持非交互标志：`npx shadcn@latest init -t vite -d`（-d 为默认值）
       - 如果 CLI 仅支持交互模式：根据 RESEARCH.md Pitfall 6 指南，在现有 Vite 项目上运行 `npx shadcn@latest init` 并选择默认值
       - 结果：生成 `components.json`、更新 `index.css`（添加 shadcn CSS 变量）、创建 `src/lib/utils.ts`
    6. 添加 shadcn/ui 组件（per UI-SPEC Design System 表）：`npx shadcn@latest add card dialog badge tooltip scroll-area separator button skeleton tabs progress alert dropdown-menu`。组件文件生成在 `frontend/src/components/ui/`。
    7. 清理 `frontend/src/main.tsx`：保留 React 19 `createRoot` 入口，移除 Vite 模板默认样式和内容。
    8. 验证：运行 `pnpm --dir frontend dev` 确认项目启动无报错（在后台启动并检查输出）。
