<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser->SSE endpoint | 浏览器通过 EventSource 消费 SSE 事件流 |
| browser->REST API | 浏览器通过 fetch() 调用 REST 端点 |
| SSE data->React state | SSE 事件数据流入 Zustand/React Query stores |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Information Disclosure | useSSE.ts | mitigate | SSE 事件数据仅存储在浏览器内存中（Zustand stores）。不持久化到 localStorage/sessionStorage。刷新页面后数据丢失——这是预期行为。如果未来需要持久化，使用 React Query 的 persist 插件。 |
| T-05-11 | Spoofing | useSSE.ts | accept | EventSource API 不支持自定义请求头（如 Authorization）。v1 本地单用户部署，无身份验证需求。 |
| T-05-12 | Tampering | api.ts | mitigate | 所有 fetch() 调用检查 `response.ok`，非 2xx 响应抛出 Error。确认端点（confirmCommand）验证响应状态。 |
| T-05-13 | Information Disclosure | eventTypes.ts:Event | accept | TypeScript 事件类型定义是静态代码——不包含运行时敏感数据。无威胁。 |
</threat_model>
