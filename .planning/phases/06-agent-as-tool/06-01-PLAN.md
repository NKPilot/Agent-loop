  <action>
    创建 AgentTool 类：

    **__tool_meta__ 属性：**
    - 返回 ToolMetadata，name=agent_meta.name, description=agent_meta.description, permission_level=SAFE
    - func_ref 指向 self.execute

    **execute(**kwargs) 方法：**
    - 接收主 Agent LLM 传来的参数（由 @agent 装饰器的参数 schema 定义）
    - 调用 _run_sub_agent() 执行子 Agent
    - 返回 AgentToolResult 序列化为字符串

    **_run_sub_agent() 内部流程：**
    1. 创建独立 EventBus（子 Agent 不污染主 EventBus）
    2. 创建独立 Session，system prompt = agent_meta.system_prompt
    3. 创建独立 ToolRegistry，复用 agent_meta.tool_registry 中的工具
    4. 创建 ToolExecutor + 必要 guards（BudgetGuard, LoopDetector, MessageValidator）
    5. 创建 LLMClient（复用主 Agent 的 config 和 api_key）
    6. 创建 ReActFSM 并运行
    7. 收集结果 → 构建 AgentToolResult

    **_extract_summary(session):**
    - summary: 取 session.messages 最后一条 assistant 回复
    - tool_calls: 从 session 事件中提取
    - token_usage: 汇总所有 step_end 的 token 数据
    - steps: session.step_count
  </action>
  <acceptance_criteria>
    - AgentTool.__tool_meta__ 返回有效的 ToolMetadata
    - AgentTool 可通过 ToolRegistry.register_meta() 注册
    - 子 Agent 创建独立 Session，不污染主 Agent session.messages
    - 返回结构化 AgentToolResult（含 summary, token_usage, session_id）
    - 子 Agent 超时时返回错误摘要
  </acceptance_criteria>
  <done>AgentTool 桥接完成，子 Agent 可被主 Agent 作为 Tool 调用</done>
