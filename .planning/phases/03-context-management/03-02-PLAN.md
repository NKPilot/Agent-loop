    在 `src/loopai/context/` 下创建 `compressor.py`，包含 `ContextCompressor` 类：

    **ContextCompressor:**
    - `__init__(self, token_counter: TokenCounter, window_size: int = 128000, threshold: float = 0.75, target: float = 0.50, preserve_rounds: int = 3)`
      - `window_size`: 上下文窗口大小（默认 128000 对应 GPT-4o）
      - `threshold`: 触发压缩的阈值比例（D-01: 0.75 = 75%）
      - `target`: 压缩目标比例（D-02: 0.50 = ~50%）
      - `preserve_rounds`: 保留的完整对话轮数（Claude's Discretion: 设为 3）

    - `async check_and_compress(self, messages: list[dict], summary_fn: Callable[[list[dict]], Awaitable[str]]) -> tuple[list[dict], bool, dict]`
      - 返回: (compressed_messages, was_compressed, metadata)
      - metadata 包含: tokens_before, tokens_after, tokens_saved, rounds_preserved, summary_message_count

      算法：
      1. 用 `self._counter.count_messages(messages)` 计算当前 token 数
      2. 如果 < `window_size * threshold`，直接返回 `(messages, False, {"tokens_before": n, "tokens_before": n, "action": "skipped"})`
      3. 需要压缩 → 调用 `_find_round_cutoff(messages)` 定位保留的消息索引
      4. 取出裁剪前的消息作为 `old_block = messages[:cutoff_idx]`
      5. 保留的消息为 `preserved = messages[cutoff_idx:]`
      6. 调用 `summary_fn(old_block)` 生成摘要（异步调用）
      7. 构造摘要消息：`{"role": "system", "content": f"[Compressed Summary] {summary}"}`
      8. 组装新消息列表：`[summary_msg] + preserved`
      9. 计算 tokens_before / tokens_after 差异
      10. 返回 `(new_messages, True, metadata)`

    - `_find_round_cutoff(self, messages: list[dict], preserve_rounds: int | None = None) -> int`
      - 从消息列表末尾反向遍历
      - 识别 "对话轮"：每条 assistant 消息（含 tool_calls）+ 随后的连续 tool 消息 = 1 轮
      - 跳过最后 `preserve_rounds` 轮，返回截断索引
      - 如果消息不够 `preserve_rounds` 轮，返回 0（不过滤任何消息）
      - 注意：system 和 user 消息（在助手首次回复之前的）应该永远不会被包含在 "轮" 中。它们在 sliding window 之外，会随着最旧内容一起被摘要。

    - `_build_summary_prompt(self, messages: list[dict]) -> str`:
      - 构建一个总结 prompt，告诉 LLM 用英文总结旧对话的关键信息和已执行的操作
      - 格式建议：
        ```
        You are an AI agent's context summarizer. Summarize the following conversation history concisely, preserving:
        1. The user's original request and key requirements
        2. All tools that were called and their results
        3. Important findings and data discovered
        4. The current state of the task
        
        Original messages:
        [{role}: {content snippet}]
        ...
        
        Summary:
        ```

    - `test_find_round_cutoff`：验证 `_find_round_cutoff` 正确保留最后 N 轮
    - `test_preserved_rounds_content`：验证保留轮次的对话消息在压缩后保持完整
    - `test_insufficient_rounds_no_cutoff`：消息轮数 < preserve_rounds 时返回索引 0

    在 `src/loopai/state_machine/guards.py` 末尾（PermissionGuard 之后）添加 `TokenGuard` 类：

    **TokenGuard:**
    - `__init__(self, token_counter: TokenCounter, window_size: int = 128000, threshold: float = 0.75)`
      - `token_counter`: TokenCounter 实例
      - `window_size`: 上下文窗口（默认 128000）
      - `threshold`: 触发压缩的阈值（D-01: 0.75）

    - `check(self, messages: list[dict]) -> tuple[str, int, int]`
      - 用 `self._counter.count_messages(messages)` 计算当前 token 数
      - 计算 `used_pct = token_count / self._window_size`
      - 返回 `(action, token_count, used_pct_pct)`：
        - `action`: "ok"（未达阈值）或 "compress"（>= 阈值）
        - `token_count`: 当前消息 token 数
        - `threshold_tokens`: 触发阈值的 token 数（即 window_size * threshold，取整）

      与 BudgetGuard 的模式一致：check() 返回状态信号，不直接修改消息列表。
      压缩的具体执行由 FSM 调用 ContextCompressor 完成。
