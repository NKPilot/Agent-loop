    **步骤 A — 创建 resilience/circuit_breaker.py**

    `CircuitState(StrEnum)`:
    - CLOSED = "closed" — 正常执行
    - OPEN = "open" — 熔断，阻止调用
    - HALF_OPEN = "half_open" — 试探调用

    `CircuitBreaker` 类：
    - `__init__(self, window_size: int = 10, failure_threshold: float = 0.5, cooldown_seconds: float = 30.0)`:
      - `self._window: dict[str, deque[bool]] = {}` — 每个 tool 一个滑动窗口，记录 True(成功)/False(失败)
      - `self._state: dict[str, CircuitState] = {}` — 每个 tool 的当前状态（默认 CLOSED）
      - `self._opened_at: dict[str, float] = {}` — 每个 tool 的打开时间戳
      - `self._probing: set[str] = set()` — 当前正在探测的 tool 集合（防止并发探测，Pitfall 1）
      - `self._lock: asyncio.Lock = asyncio.Lock()` — 保护半开状态转换的并发安全
      - 存储 window_size, failure_threshold, cooldown_seconds

    - `check(self, tool_name: str) -> tuple[bool, CircuitState]`:
      - 半开互斥逻辑：
        1. 如果 `tool_name in self._probing`，返回 `(False, CircuitState.OPEN)` — 已有探测进行中
        2. 如果 `tool_name in self._state` 且 `self._state[tool_name] == CircuitState.OPEN`：
           - 计算 `elapsed = time.monotonic() - self._opened_at[tool_name]`
           - 如果 `elapsed >= self.cooldown_seconds`：
             - 将 `self._state[tool_name] = CircuitState.HALF_OPEN`
             - `self._probing.add(tool_name)` — 标记为正在探测
             - 返回 `(True, CircuitState.HALF_OPEN)` — 允许一次试探
           - 否则返回 `(False, CircuitState.OPEN)` — 仍处于冷却期
      - 如果工具没有记录的状态或状态不是 OPEN，返回 `(True, state)`（CLOSED 或 HALF_OPEN）

    - `async record(self, tool_name: str, success: bool, bus: EventBus | None = None)`:
      - 初始化：如果 `tool_name not in self._window`，创建 `deque(maxlen=self.window_size)`
      - 追加结果：`self._window[tool_name].append(success)`
      - 计算失败率：`failures = sum(1 for r in self._window[tool_name] if not r); rate = failures / len(window)`
      - 从 `self._probing` 移除（如果存在）
      - 获取当前状态：`state = self._state.get(tool_name, CircuitState.CLOSED)`

      - **HALF_OPEN 状态处理**（使用 `async with self._lock` 保护并发）：
        - 如果成功：`self._state[tool_name] = CircuitState.CLOSED; previous = "half_open"; new = "closed"`
          - 发布 CircuitClosed 事件
        - 如果失败：`self._state[tool_name] = CircuitState.OPEN; self._opened_at[tool_name] = time.monotonic(); previous = "half_open"; new = "open"`
          - 发布 CircuitOpened 事件

      - **CLOSED 状态处理**：
        - 如果 `rate > self.failure_threshold`：
          - `self._state[tool_name] = CircuitState.OPEN; self._opened_at[tool_name] = time.monotonic()`
          - 发布 CircuitOpened 事件
        - 否则：保持 CLOSED，不发布事件

      - **事件发布格式**（如果 bus 不为 None）：
        - CircuitOpened: `{"event_type": "circuit_opened", "session_id": session_id, "tool_name": tool_name, "failure_rate": rate, "window_size": self.window_size, "previous_state": previous, "new_state": "open", "timestamp": ...}`
        - CircuitClosed: `{"event_type": "circuit_closed", "session_id": session_id, "tool_name": tool_name, "previous_state": previous, "new_state": "closed", "timestamp": ...}`

    - `async def record_with_session(self, tool_name: str, success: bool, session_id: str, bus: EventBus | None = None)`:
      - 调用 `record`，但传递 session_id 到事件数据中

    - `get_open_tools(self) -> set[str]`:
      - 返回所有 `self._state[tool] == CircuitState.OPEN` 的 tool 名称集合（用于通知 ToolRegistry 过滤 Schema）

    - `get_tool_state(self, tool_name: str) -> CircuitState`:
      - 返回 `self._state.get(tool_name, CircuitState.CLOSED)`

    - `reset_tool(self, tool_name: str)`:
      - 删除 `tool_name` 的 window、state、opened_at、probing 记录
