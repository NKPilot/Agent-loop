<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 工具函数 -> ToolExecutor | 工具函数来自开发者注册，但参数来自 LLM（不可信），executor 在此边界做 Pydantic 校验 |
| ToolExecutor -> 系统资源 | 工具执行可访问文件系统/网络/进程，executor 超时机制限制资源占用 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | decorator.py | mitigate | Pydantic 自动校验所有 LLM 传入的参数——在调用工具函数前完成 Marshal+Validate，校验失败返回结构化错误而不调用函数体 |
| T-02-02 | Elevation of Privilege | executor.py | mitigate | FatalError（OOM/SystemExit）直接 re-raise 终止会话，防止工具通过耗尽资源逃逸；超时机制（asyncio.wait_for）限制每个工具的执行时间 |
| T-02-03 | Denial of Service | executor.py | mitigate | 每个工具有独立 timeout（D-07），默认 30s；重试有上限（max_attempts），max_delay 封顶 60s；防止无限重试耗尽资源 |
| T-02-04 | Information Disclosure | types.py | mitigate | ToolMetadata 的 func_ref 使用 Field(exclude=True) 排除序列化，防止工具函数引用泄露到 JSON Schema 或日志 |
</threat_model>
