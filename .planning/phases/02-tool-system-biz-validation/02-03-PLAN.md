       ```
       for each tool_call in last_assistant_message.tool_calls:
           tool_name = tool_call["name"]
           tool_args = tool_call["arguments"]
           tool_call_id = tool_call["tool_call_id"]
           
           # Step 1: 查找工具
           metadata = self.registry.get(tool_name)
           if metadata is None:
               session.add_message("tool", content="[SYSTEM] 工具 '{tool_name}' 未注册", tool_call_id=tool_call_id)
               continue
           
           # Step 2: 权限检查（permission_level 从 metadata 获取）
           should_proceed, action = await self.permission_guard.check(
               tool_name, tool_args, metadata.permission_level,
               session.session_id, session.step_count
           )
           
           if action == "confirm_required":
               # PermissionGuard 内部已发布 confirmation_required 事件
               # 阻塞等待用户响应或超时
               # should_proceed 在 check() 内部等待 asyncio.Event
               # 返回时 should_proceed=True 表示用户确认，False 表示拒绝/超时
               pass
           
           if not should_proceed:
               reason = "用户拒绝" if action == "user_denied" else "确认超时"
               session.add_message("tool", content=f"[SYSTEM] 操作被{reason}：{tool_name}", tool_call_id=tool_call_id)
               continue
           
           # Step 3: 执行工具
           result = await self.executor.execute(tool_name, tool_args)
           
           # Step 4: 发布 ToolResult 事件
           await self.bus.publish("tool_result", {
               "event_type": "tool_result",
               "session_id": session.session_id,
               "step_num": session.step_count,
               "tool_name": tool_name,
               "tool_call_id": tool_call_id,
               "result": str(result.data) if result.data else "",
               "is_error": result.status == "error",
               "duration_ms": result.duration_ms,
           })
           
           # Step 5: 注入 tool_result 到消息列表
           session.add_message("tool", content=str(result.data) if result.data else result.error or "", tool_call_id=tool_call_id)
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI 用户输入 -> PermissionGuard | 用户通过 stdin 输入 y/n 响应，通过 PermissionGuard.respond() 传递——这是信任边界，因为 CLI 渲染器在用户空间运行 |
| FSM -> ToolExecutor | FSM 从 LLM 消息中提取 tool_name/tool_args，传递给 executor——这是不可信数据经过 Pydantic 校验的边界 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Spoofing | cli_renderer.py | mitigate | confirmation_response 事件包含 confirmation_id 用于匹配——防止重放或伪造的响应；PermissionGuard 通过 confirmation_id 查找对应的 asyncio.Event |
| T-02-12 | Elevation of Privilege | fsm.py | mitigate | FSM._handle_act() 始终通过 PermissionGuard.check() 检查权限——不可能绕过确认机制直接执行 DANGEROUS 命令（permission_level 在工具注册时设定，不可被 LLM 覆盖） |
| T-02-13 | Denial of Service | fsm.py | mitigate | confirmation_timeout（默认 120s）防止无限等待用户输入导致 Agent 会话挂起；超时后自动拒绝并注入 tool_result 说明原因 |
| T-02-14 | Information Disclosure | config.py | mitigate | 新增的 tool_working_dir, tool_timeout, confirmation_timeout 字段不包含密钥——verbose 模式可安全打印 |
</threat_model>
