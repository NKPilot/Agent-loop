       返回 stdout 字符串
    
    3. **disk_find** (`@tool(name="disk.find", description="按大小/类型筛选文件。查找大于指定大小的文件或匹配模式的文件。", permission_level=PermissionLevel.SAFE, timeout=30.0, tags=["disk", "diagnostic"])`):
       异步函数，接受 `directory: str = "/tmp/loopai-demo"`, `min_size: str = "+10M"`（如 "+100M", "+1G"）, `file_type: str = "f"`（f=文件, d=目录）
       内部调用 `subprocess.run(["find", directory, "-type", file_type, "-size", min_size, "-exec", "ls", "-lh", "{}", ";"], capture_output=True, text=True, timeout=30, shell=False)`
       限制 directory 必须在 working_dir 内
       返回 stdout 字符串
    
    4. **disk_rm** (`@tool(name="disk.rm", description="删除指定的文件或目录。**危险操作**——执行前必须经过确认。", permission_level=PermissionLevel.DANGEROUS, timeout=30.0, tags=["disk", "cleanup"])`):
       异步函数，接受 `target: str`（要删除的文件或目录路径）和 `recursive: bool = False` 参数
       内部调用分类器检查 target 是否在 working_dir 内
       如果 target 不在 working_dir 内 → 抛出 GuardViolationError("清理操作仅限于沙箱目录内")
       调用 `subprocess.run(["rm", "-rf" if recursive else "-f", target], capture_output=True, text=True, timeout=30, shell=False)`
       返回 stdout 字符串（通常为空）+ "已删除 {target}" 确认信息

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 用户自然语言 -> Agent | 用户通过 prompt 和交互式 y/n 输入与 agent 通信——不可信输入边界 |
| Agent -> disk_rm 工具 | disk_rm 执行真正的文件删除——必须有路径限制防止逃逸 |
| disk_du/disk_find -> 文件系统 | 诊断工具只读——安全，但需限制 directory 范围防止信息泄露 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-15 | Elevation of Privilege | disk_tools.py disk_rm | mitigate | disk_rm 硬编码检查 target 必须在 working_dir（/tmp/loopai-demo）内；路径不在沙箱内时抛出 GuardViolationError，绝不执行 |
| T-02-16 | Tampering | disk_tools.py disk_du, disk_find | mitigate | directory 参数通过 Pydantic validator 限制在 working_dir 内；LLM 不能通过参数注入使工具访问沙箱外的文件系统 |
| T-02-17 | Information Disclosure | disk_tools.py disk_df | accept | disk_df 显示真实系统磁盘信息——这是设计意图（D-16："df 展示真实系统数据"），提供诊断上下文。不包含 PII，低风险 |
| T-02-18 | Denial of Service | setup_demo_scenario.sh | mitigate | dd 创建的文件大小有上限（总场景 ~1GB）；find/du 命令有 timeout 限制（30s）；预设文件数量有限 |
</threat_model>
