## 工具注册与执行管线

### @tool 装饰器配置范围

| 选项 | 描述 | 选中 |
|------|------|------|
| 最小配置 | @tool(name, description) — 其他从 type hints 自动推导 | |
| 完整配置 | @tool(name, description, permission_level, timeout, retry, tags) — 元数据驱动 | ✓ |

### 参数校验方式

| 选项 | 描述 | 选中 |
|------|------|------|
| Pydantic 自动校验 | type hints 构造模型，自动 Marshal+Validate，校验失败返回结构化错误 | ✓ |
| 工具函数内自查 | 管线只做类型转换，校验由工具函数自行处理，更灵活 | |

### JSON Schema 生成方式

| 选项 | 描述 | 选中 |
|------|------|------|
| Python→JSON Schema 映射表 | 维护 type mapping table，覆盖绝大多数场景，无额外依赖 | ✓ |
| Pydantic TypeAdapter | 处理所有边缘类型，更完整但可能产生 LLM 不友好的字段 | |

### ToolRegistry 组织方式

| 选项 | 描述 | 选中 |
|------|------|------|
| 全局单例注册表 | 一个全局实例，按名称索引，适合单 agent | |
| 实例注册表+命名空间 | 多实例、命名空间（'bash.ls', 'disk.du'），支持未来多 agent | ✓ |

### 工具结果处理

| 选项 | 描述 | 选中 |
|------|------|------|
| ToolResult 标准化包装 | 统一包装为 ToolResult(status, data, error, duration_ms, truncated) | ✓ |
| 保留原生输出 | 工具返回什么就注入什么，更灵活 | |

### sync/async 支持

| 选项 | 描述 | 选中 |
|------|------|------|
| 两者都支持 | 自动检测——async 用 await，sync 用 asyncio.to_thread() | ✓ |
| 仅异步 | 所有工具必须是 async 函数，统一模型但 subprocess 天然 sync | |

### 超时处理

| 选项 | 描述 | 选中 |
|------|------|------|
| 装饰器默认+可覆盖 | @tool(timeout=60)，全局默认 30s，Bash 默认 60s | ✓ |
| 全局统一超时 | 所有工具共用一个值，简单但不灵活 | |

## Bash 安全与权限分级

### 危险命令判定

| 选项 | 描述 | 选中 |
|------|------|------|
| 黑名单+模式匹配 | 危险模式列表（rm -rf, dd, mkfs...），正则匹配 | |
| 白名单+默认拒绝 | 只允许预注册的安全命令，不在白名单拒绝 | |
| 两者结合 | 白名单为底自动放行，灰名单 moderate，黑名单强制 dangerous | ✓ |

### 确认机制

| 选项 | 描述 | 选中 |
|------|------|------|
| 事件驱动暂停 | EventBus confirmation_required 事件，FSM 暂停，CLI 等待 y/n | ✓ |
| 同步 CLI input() | 直接调用 input()，简单但破坏 EventBus 解耦 | |

### 权限级别标准

| 选项 | 描述 | 选中 |
|------|------|------|
| 按操作类型 | safe=只读, moderate=可逆写, dangerous=不可逆写+系统级 | |
| 按影响范围 | 同一命令按路径不同可属不同级别——更精细的安全模型 | ✓ |

## 错误分类与重试策略

### 错误分类方法

| 选项 | 描述 | 选中 |
|------|------|------|
| 异常类型映射 | 每种异常自动映射到 Transient/ToolExecution/GuardViolation/Fatal | ✓ |
| 工具函数自行分类 | 工具作者通过参数指定类别，最大控制权但可能不一致 | |

### 重试配置方式

| 选项 | 描述 | 选中 |
|------|------|------|
| 装饰器可配置 | @tool(retry=RetryConfig(...))，有全局默认值，可按需覆盖 | ✓ |
| 全局统一配置 | 所有工具共享同一套参数，简单但不灵活 | |

### 重试触发范围

| 选项 | 描述 | 选中 |
|------|------|------|
| 仅 TransientError | 只有明确可恢复的错误（超时、网络、ConnectionError）触发重试 | ✓ |
| TransientError + ToolExecutionError | 部分执行错误可能因临时状态，重试可能解决 | |
