    **PermissionGuard 设计（D-08, D-09）：**

    ```python
    class PermissionGuard:
        """权限守卫——在工具执行前检查权限级别。
        
        SAFE/MODERATE 命令直接放行。DANGEROUS 命令通过 EventBus 发出
        confirmation_required 事件，阻塞等待用户确认或拒绝。
        """
        
        def __init__(self, bus: EventBus, confirmation_timeout: float = 120.0):
            self._bus = bus
            self._confirmation_timeout = confirmation_timeout
            self._pending_confirmations: dict[str, asyncio.Event] = {}
            self._confirmation_results: dict[str, bool] = {}
        
        async def check(
            self, 
            tool_name: str, 
            tool_args: dict, 
            permission_level: PermissionLevel,
            session_id: str,
            step_num: int,
        ) -> tuple[bool, str]:
            """检查此工具调用是否需要确认。
            
            Returns:
                (should_proceed, action)
                action: "allow" | "confirm_required" | "user_denied" | "timeout"
            """
            ...
        
        def respond(self, confirmation_id: str, approved: bool) -> None:
            """用户对确认请求的响应。由 CLI 消费者调用。"""
            ...
    ```

    **实现逻辑：**
    1. SAFE 或 MODERATE → 直接返回 (True, "allow")
    2. DANGEROUS → 生成 confirmation_id（`f"{session_id}_{tool_name}_{step_num}"`）
    3. 发布 EventBus 事件 `"confirmation_required"`：{event_type, session_id, step_num, confirmation_id, tool_name, tool_args, permission_level, reason, timestamp}
    4. 创建 asyncio.Event，记录到 `_pending_confirmations`
    5. 阻塞等待事件或超时：`await asyncio.wait_for(event.wait(), timeout=self._confirmation_timeout)`
    6. 超时 → 返回 (False, "timeout")，发布 `"confirmation_timeout"` 事件
    7. 用户确认 → 返回 (True, "allow")
    8. 用户拒绝 → 返回 (False, "user_denied")

    **respond() 方法：**
    - 通过 confirmation_id 查找对应 asyncio.Event
    - 存储结果到 `_confirmation_results`
    - 设置 Event（唤醒阻塞的 check() 调用）

    测试用例追加到 `tests/test_bash_tool.py`。

    注意：PermissionGuard 不依赖 CommandClassifier 直接分类——它接收已分类的 permission_level。分类逻辑在 BashTool 中完成，PermissionGuard 只根据级别决定行为。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM 工具参数 -> BashTool | LLM 生成的命令字符串不可信，必须通过 CommandClassifier + shell 元字符检测 |
| BashTool -> subprocess | 命令执行边界——shell=False 防止注入，数组参数防止空格拆分攻击，timeout 防止资源耗尽 |
| BashTool -> 文件系统 | 命令输出可访问文件系统——路径范围检测限制危险操作范围 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Spoofing | bash.py | mitigate | shell=False 防止命令注入；数组参数使用 shlex.split() 安全解析；shell 元字符（\|, ;, &, $(), ``）直接拒绝 |
| T-02-06 | Tampering | bash.py | mitigate | LLM 提供的命令字符串在分类前通过 CommandClassifier 白名单/黑名单检查；subprocess.run 使用绝对路径 cwd 限定工作目录 |
| T-02-07 | Elevation of Privilege | command_classifier.py | mitigate | 路径感知升级：操作 /etc//dev//sys//proc//boot 等系统路径的命令自动升级至少一级（SAFE->MODERATE, MODERATE->DANGEROUS）；chmod 777 /、输出重定向到设备文件等危险模式硬编码检测 |
| T-02-08 | Denial of Service | bash.py | mitigate | 每个命令有独立 timeout（默认 60s）；输出上限 100KB 截断防止内存爆炸；subprocess.run 使用 capture_output=True 限制输出 |
| T-02-09 | Repudiation | guards.py | mitigate | confirmation_required 和 confirmation_response 事件通过 EventBus 发布->JSONL 记录，提供完整审计轨迹：谁请求确认、用户响应是什么、何时发生 |
| T-02-10 | Information Disclosure | bash.py | mitigate | 命令输出在截断前与截断后均通过 ToolResult 返回；超长输出写入溢出文件（Phase 3 处理）而非静默丢弃 |
</threat_model>
